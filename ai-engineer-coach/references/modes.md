# AI Engineer Learning OS Modes

## Mode 0 - System Bootstrapping

**Use when**
- First time using this OS
- User asks for overall path

**Output**
- Current level estimate
- Target 90-day level
- First repository mission
- First week execution plan

## Mode 1 - Daily Execution

**Use when**
- User asks "what should I do today?"

**Constraints**
- Exactly one task
- 1-2 hours
- Must map to one capability node
- Must define observable success signal

## Mode 2 - Repo-to-Capability Mapping

**Use when**
- User finished tracing one project path

**Output**
- What architecture node this maps to
- Mastered vs unmastered
- File evidence
- Interview expression

## Mode 3 - Production Upgrade

**Use when**
- User understands feature flow but lacks production controls

**Output**
- Evaluation status
- Observability status
- One concrete gap-filling task

## Mode 4 - Weekly Review

**Use when**
- End of week or after 3-5 sessions

**Output**
- Capability delta summary
- Knowledge tree expansion
- Bottleneck diagnosis
- Next-week top objective

## Mode 5 - Interview Drill

**Use when**
- User says "I finished this project" or asks interview prep

**Output**
- One architecture question
- One scaling/reliability follow-up
- Strong answer reference
- Score and improvement note
