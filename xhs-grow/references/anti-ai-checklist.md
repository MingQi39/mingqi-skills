# 人味自检清单（发布前 MUST 全过）

任何笔记完成后逐条检查。**一条不过则重写。**

## 一、词汇红线

- [ ] 无「姐妹们」「家人们」「宝子」「绝绝子」「yyds」
- [ ] 无「干货」「必看」「码住」「建议收藏」
- [ ] 无「今天给大家分享」「希望对大家有帮助」
- [ ] 无「首先…其次…最后…」「综上所述」
- [ ] 无 Step 1/2 编号（除非真教流程）
- [ ] 标题 ≤ 13 个汉字

## 二、结构红线

- [ ] 开头是具体场景或情绪
- [ ] 结尾无总结、升华、号召
- [ ] 至少 1 处生活细节「跑题」
- [ ] 数据夹在叙述里，非 bullet 列表
- [ ] 全角标点；引述不加引号；数字/英文两侧半角空格

## 二·五、Sequel 自检（承接上条爆款时）

见 [`sequel-strategy.md`](sequel-strategy.md)：

- [ ] 类型与上条爆款一致（通常故事体）
- [ ] 开篇不是「评论区问太多了」
- [ ] 与上条无连续 8 字以上相同段落
- [ ] 中段有评论钩子

## 三、人味必备（≥ 3 项）

- [ ] 自我暴露（我也搞不懂/我也烦）
- [ ] 时间锚点
- [ ] 口语虚词
- [ ] 情绪波动
- [ ] 具体细节（数字/对话/物件）
- [ ] 符合 `account-persona.md` 叙事红线

## 四、信息红线

- [ ] 不确定的数据有「建议自行核对」
- [ ] 无产品/机构背书
- [ ] 无绝对化用语
- [ ] 批评用「对 X 人群可能不合适」

## 五、幽默（见 `humor.md`）

- [ ] 笑点不落在读者/弱势群体身上
- [ ] 结尾平收，非抖包袱

## 六、最终三问

1. 发闺蜜微信会不会奇怪？
2. 有没有现实里不会说的话？
3. 像不像在「运营」一个号？

任一「是」→ 重写。

## 七、发布形态

见 [`publish-formats.md`](publish-formats.md)：配图模式 vs 文字笔记（§2 纯文本 + §2b 加粗清单）。
