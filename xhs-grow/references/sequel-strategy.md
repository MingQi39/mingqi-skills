# Sequel 承接策略（通用版）

> 上一条笔记爆了或评论集中问同一类问题时，下一条怎么接。**承接话题，不换写法、不复用爆款段落。**

---

## 核心结论

**Sequel = 新故事 + 老问题。** 可以接评论区话题，不可以：

- 换体裁（故事爆款后立刻发纯教程 / 横评）
- 开篇「评论区问太多了」「接上条」
- 复制上条连续 8 字以上的段落

---

## 做对了什么（应复制）

| 维度 | 做法 |
|---|---|
| 类型 | 延续已验证的内容类型（通常是故事 / 避坑） |
| 开头 | 新画面、新场景，陌生人 3 秒能看懂 |
| 信息 | 干货夹在叙事里，不列表罗列 |
| 形态 | 与上条数据更好的发布形态一致（纯字 vs 配图） |
| 独特性 | 新细节、新对话，不可从上帖照抄 |

---

## 做错了什么（禁止）

| 踩雷 | 后果 |
|---|---|
| 类型降级（故事 → 纯教程 / 横评） | 停留短、评论少，流量池封顶 |
| 段落复用爆款原文 | 老读者划走，平台可能判重复 |
| 开头依赖上条 | 新流量第一句就懵 |
| 无中段互动钩子 | 评论个位数，CES 上不去 |

---

## 写稿铁律（MUST）

1. **MUST** 用与上条爆款相同的**内容类型**（见 `templates/`）
2. **MUST** 把实操步骤 / 对比表 **塞进叙事**，不直接发教程体
3. **MUST** 自我重复检测：与上条正文比对，无连续 8 字以上相同
4. **禁止** 开篇「评论区问太多了」；第一句必须是 **新画面**
5. 优先 **文字笔记** 测文案；有实拍封面再切配图模式

---

## 选题优先级

1. 上条评论反复出现的问题（比大盘热搜更准）
2. 检索热度高 + 与人设契合的题
3. 库存选题库里未发的题（见 `topic-pool-template.md`）
