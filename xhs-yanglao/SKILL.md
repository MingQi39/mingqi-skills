# 小红书「我妈的养老地图」内容生产 skill

## 触发场景

当用户提到下面任一情况,自动激活本 skill:

- 写一条养老笔记 / 帮我妈选养老地 / 候鸟养老
- 我现在做到哪了 / 下一步是什么 / 该发什么了
- 写小红书 / 写笔记(在养老地图账号上下文中)
- 想个标题 / 出张封面 / 优化文案(养老地图相关)
- 纯字发 / 文字笔记 / 不要配图 / 先不拍照
- 热门话题 / 什么选题火 / 帮我选题

4. [`references/algorithm.md`](references/algorithm.md) — 平台流量机制 + CES 公式 + 涨粉指标 + 红线(决定怎么写才能涨粉)

**要推荐选题 / 排下一条 / 用户未指定具体题号时**,额外 MUST:

7. [`references/hot-topics.md`](references/hot-topics.md) — 热门检索 SOP + 当前热度排行 + 排期修正

并执行 **WebSearch**(≥3 组检索词,见 hot-topics.md §二)。`hot-topics.md` 超过 14 天未更新 → 检索后 **append 新快照到该文件** 再推荐。

用户要**纯字 / 文字笔记**时,额外读 [`references/publish-formats.md`](references/publish-formats.md) — 发布形态、输出第 4 节怎么写、数据复盘格式。

不读 algorithm.md = 写出来的内容能看但不能涨粉(没埋互动钩子、SEO 缺位、踩平台红线)。

## 用户进度跟踪(CRITICAL)

每次会话开始 / 用户问"做到哪了"/ 完成一个里程碑,**MUST**:

1. 读 [`state/progress.md`](state/progress.md) 看当前阶段
2. 用「日历日期 + 阶段名 + 下一动作」回答
3. 用户完成动作后,**主动 append 一条记录到 `state/progress.md`**(用日期+动作+备注的格式)

进度文件是 single source of truth。我说什么不算,文件里记的才算。

## Git 自动同步(CRITICAL)

本仓库是 skill 的 single source of truth,**每次在本仓库内增删改任何文件后,MUST 立即 commit 并 push 到 `origin`**,不要等用户提醒,也不要只改本地不落库。

### 何时触发

- 写完/改完笔记相关产出并写入文件时
- 更新 `state/progress.md` 时
- 修改 `references/`、`templates/`、`examples/`、`SKILL.md` 等任意路径时
- 会话结束前若仍有未提交的改动,**必须先提交再结束**

纯对话、未落盘到本仓库的文件 → 不触发。

### 执行步骤(每次改动后按序做)

1. 在仓库根目录执行 `git status`、`git diff`(含 staged)、`git log -3 --oneline`,了解当前变更与近期 commit 风格
2. **不要**提交 `.env`、密钥、token、私钥等敏感文件;若误改到则还原并警告用户
3. `git add` 仅包含本次相关的变更(通常 `git add -A` 即可,但排除不应入库的文件)
4. 用 HEREDOC 写 commit message,1-2 句说明「为什么」,格式示例:`YYYY-MM-DD: 更新 progress — Day 3 养号完成` 或 `产出第 N 条笔记草稿《标题》`
5. `git commit`(**禁止** `--no-verify` / `--no-gpg-sign`,除非用户明确要求)
6. `git push`(当前分支跟踪 `origin` 时直接 `git push`;未设置 upstream 则用 `git push -u origin HEAD`)
7. 再跑 `git status` 确认 working tree clean 且已推送

### 禁止事项

- 不 `git push --force` 到 `main`/`master`(除非用户明确要求)
- 不 `git commit --amend`(除非用户明确要求且未 push)
- 不修改 `git config`
- commit 失败(含 hook 拒绝)时:修问题后**新建 commit**,不要 amend 失败的提交

## 内容生产流程

### Step 0:热门选题检索(CRITICAL,先于写稿)

**触发**:用户问「该发什么」「下一条」「想选题」、或写笔记但未指定 # 号 / 具体标题。

**禁止**:只翻 `topic-pool.md` 按 Day 表机械推荐。

**MUST 顺序**:

1. 读 [`references/hot-topics.md`](references/hot-topics.md) §五排行 + [`state/progress.md`](state/progress.md) 上条笔记评论主题
2. **WebSearch** ≥3 组(赛道大盘 / 账号垂直 / 政策或城市长尾),更新认知;若 §四快照 **>14 天** → 把新结论 append 进 hot-topics.md
3. 用 hot-topics.md **§三评分表** 给 2–3 个候选打分,只推荐 **总分 ≥14** 的题
4. 向用户输出时**每条带** `热度依据: …` + 建议标题(≤13 字) + 类型;等用户确认后再进入 Step 1

**优先 sequel**:上条笔记评论里反复出现的问题(如 #19 的医保、三城、湿度) > 库存里未验证的城市冷题。

**Sequel 类型锁定(CRITICAL,2026-06-08 实测)**:#19(8300 阅)后 #10、#1 各 ~100 阅 — 话题对、**写法错**。承接评论题时:

- **MUST** 仍用 **故事/避坑**(`templates/story.md`),把实操步骤/城市对比 **塞进叙事**,不直接发教程/横评
- **MUST** 读 `sequel-strategy.md` §自我重复检测;与 #19 连续 8 字以上相同段落 → 重写
- **禁止** 开篇「评论区问太多了」;第一句必须是 **新画面**
- Phase 2 在出现第二条 ≥1000 阅前:**连续 2 条 MUST 故事体**

### Step 1:确认选题类型

四种类型,选其一(没说明就问;**已在 Step 0 确认题号则直接对应模板**):

| 类型 | 适用场景 | 模板 |
|---|---|---|
| **故事/避坑** | 第一人称叙事,带情绪和细节,涨粉主力;**#19 已验证,Phase 2 默认** | [`templates/story.md`](templates/story.md) |
| **对比** | 多城市/多方案横向对比,收藏率高;**勿作 #19 后 immediate sequel** | [`templates/compare.md`](templates/compare.md) |
| **实操** | 流程/清单/政策办理;**医保等热题也须故事包装,勿纯教程首发** | [`templates/tutorial.md`](templates/tutorial.md) |
| **吐槽/复盘** | 短帖,情绪化,易出爆款 | [`templates/rant.md`](templates/rant.md) |

### Step 2:核对真实信息(MUST)

如果选题涉及政策/数据/具体地点,**先问用户有没有真实素材**。如果没有:

- 政策类:**拒绝凭空生成**,告诉用户先查官方源(社保局/医保局官网),贴回来再写
- 数据类(房租/气温等):可以生成参考值,但**必须在文末「核对清单」里列出来让用户确认**
- 故事类:可以编但要像真的(具体时间/地点/对话/细节)

**绝不**让用户拿着 LLM 编的政策数据去发笔记——平台监管和粉丝信任都会出事。

### Step 3:按模板生产

读对应 `templates/<类型>.md`,严格按里面的结构和约束生成。

### Step 4:人味自检

生产完成后,用 `references/anti-ai-checklist.md` 自查一遍。**任何一条不过,重写**。

### Step 5:配图产出(MUST,与正文同次完成)

完整规则:[`references/image-pipeline.md`](references/image-pipeline.md) + [`references/publish-formats.md`](references/publish-formats.md)。

用户明确 **纯字 / 文字笔记 / 不要配图** → **跳过本节**,输出改「发布形态」操作提醒,**不跑**配图脚本。

**Phase 2 默认形态(2026-06-08)**:账号尚无第二条 ≥1000 阅、#19 纯字 CTR 19.6% 而 #10/#1 脚本配图各 ~100 阅 → **未特别声明时,默认文字笔记**,不跑配图脚本。用户有实拍封面时再切「1 实拍 + 纯字」。

否则(用户明确要求配图 / 已有实拍封面) → **配图模式**,AI **必须**:

1. 将正文写入 `drafts/note<题号>-<简称>.md`(可选 YAML `memo:` 块,供脚本读)
2. 在仓库根目录执行 **`python3 tools/gen_note_assets.py --note <题号> --date YYYY-MM-DD`**(发笔记当天;省略 `--date` 用今天;有草稿 YAML 则加 `--draft …`)
3. 确认 `assets/usable/` 下 PNG 已生成;若失败则修脚本/依赖后重跑,**不得**只给用户「拍什么」文字清单了事
4. 向用户说明 **图 1 封面** = 手机实拍 `养老-封面通用.jpg`(茶杯+笔记本,2 分钟,多条复用);若用户暂无法拍,写明可改发文字笔记或先发备忘录图试 A/B

#### 人味配图标准(MUST)

| 用 | 禁用 |
|---|---|
| `tools/gen_note_assets.py` → **纯文本白卡片** PNG(定稿标准见 image-pipeline.md) | `gen_note_images.py`、假 iOS 备忘录头、卡通图 |
| 用户实拍封面、真实 App 截屏(实操) | AI 插画封面、Canva 海报、伪造聊天/官方界面 |
| 对比表/清单 → 备忘录第 2 张起 | 把大表 P 在封面上 |
| 备忘录标题短而直白、正文可多字 | 标题含「替我整理」「我记的」；正文含 **#19/#11 等题号**(见 image-pipeline.md) |

配图文案写在 [`tools/memo_presets.py`](tools/memo_presets.py)，**出图后提醒用户看图改字**，改完重跑 `gen_note_assets.py`。

**实操类(#10 等)**:备忘录示意 + **必须提醒**用户补「国家医保服务平台」等 **真实操作截屏**(打码)。

**#19 已验证**:纯字 CTR 19.6%;**#10/#1 脚本配图 ~100 阅** — 权重恢复前优先纯字。有实拍封面时再 **实拍 1 张 + 备忘录 1–2 张**。

#### 输出第 4 节格式(配图模式)

写 **「配图文件」**,不是泛泛「配图建议」:

- 上传顺序列表 + 每个文件的仓库路径(如 `assets/usable/note11-memo-climate.png`)
- 封面实拍要求(链到 image-pipeline §封面,一句话)
- 须用户自备的图(如实操 App 截屏)
- 可选:抄录到手机备忘录自截屏,更有人味

## 输出格式

### 选题推荐任务(Step 0 完成后的简报)

直接输出,不写前言:

1. **检索摘要**(2–4 条:本月平台趋势 + 本账号评论信号 + 高意图搜索词)
2. **推荐 1–3 个选题**(题号 + ≤13 字标题 + 类型 + 总分 + **热度依据**)
3. **不建议近期发的 1–2 个**(来自库存,说明为什么不热)
4. **排期建议**(若用户在做 30 天计划,对照 hot-topics.md §六)

### 笔记生产任务(Step 1–6 完成后)

直接按这 6 节输出,不写前言后记:

1. **5 个标题备选**(**全部 ≤ 13 个汉字**,不写到截断;首屏只显示 13 字,长了就被「...」吃掉)。不用感叹号,不用"必看/绝绝子/家人们",不用副标题分号(像「跑了 8 个城市,我妈最后选了一个她原本看不上的」这种 22 字 = ❌ 截断)。
2. **正文**(400-600 字,**纯文本可直接粘贴**;全角标点,**禁止 `**` 星号与任何 Markdown**;见下节)
2b. **加粗清单**(3–6 条,每条为正文中的原文字符串;用户 App 内选中后点 B,见「重点加粗」)
3. **标签**(5-8 个)
4. **配图文件**或**发布形态**(纯字:文字笔记操作提醒;否则:**已生成的 PNG 路径** + 上传顺序 + 封面实拍说明 + 用户须补截屏)
5. **评论区互动话术**(3 条,自己用小号引导)
6. **核对清单**(列出所有不确定的具体数字/政策/地点,让用户人工查证)

## 排版与标点规范(MUST,所有正文都要严格按这个,不要再自己发挥)

**这一节是从用户实际改稿里反推出的硬约束。AI 输出正文时,先按下面的规则写,然后逐条比对一遍才能交。**

### 标点(全部用全角)

| 用 | 不用 |
|---|---|
| `,`(全角逗号,U+FF0C) | `,`(半角)、` `(空格断句) |
| `。`(全角句号) | `.` |
| `;`(全角分号,U+FF1B) | `;`(半角)、`;`(半角中文风) |
| `:`(全角冒号) | `:` |
| `?`(全角问号) | `?` |
| `!`(全角感叹号),非必要不用 | `!` |
| `——`(中文破折号,两个全角) | `--`、`-` |
| `……`(中文省略号) | `...` |

### 引号:多数情况下不用

引述别人说的话**直接写,不加引号**:

- ✅ 我妈说你让我去南方,我去威海干嘛,这跟回家有啥区别——
- ❌ 我妈说「你让我去南方,我去威海干嘛」
- ❌ 我妈说"你让我去南方,我去威海干嘛"

只有引一个**短词 / 中介话术 / 网络词**时,才用直角引号「」,不用 ""。

### 数字 / 英文 / 单位两侧:加半角空格

- ✅ 替她跑了 8 个城市
- ✅ 淡季月租也得三千五往上 / 实际走 23 分钟
- ✅ 5 分钟到沙滩
- ❌ 替她跑了8个城市
- ❌ 实际走23分钟

中文-中文之间不加空格,中文-数字 / 中文-英文之间加一个半角空格(这是用户的固定习惯)。

### 段落与换行

- 段内**不为节奏强行断行**。一段就是一段连续的话,该用句号 / 分号 / 破折号串就串起来。
- **每段 1-3 句,3-5 行**,一篇 5-8 段。
- 段落之间**留一个空行**(回车 + 空行 + 回车),AI 输出时**就用一个空行**,不要连留两三个空行。
- 段落之间**不要标号、不要 bullet、不要小标题**。
- 短句单独成段是可以的(例:「我也没问她为啥。」单独一段),但前提是这一句**有重量**,不是凑节奏。

### 一个完整对照样本(从用户 #19 改稿截取,作为「就长这样」的参照)

```
三亚最贵。我看的那几个小区,淡季月租也得三千五往上,旺季翻倍我都没敢跟她细讲;中介管隔条马路 + 楼后面那种叫望海,我学到一个新词。

万宁是中介推得最猛的一个,海景房月租两千。我导航量了一下步行 5 分钟到沙滩,实际走 23 分钟,中间两个红绿灯;我说怎么差这么多,他说他们一般按车程算。行,你说啥就是啥。
```

注意:
- 全部用全角逗号 / 句号 / 分号
- 引述「望海」「步行 5 分钟到沙滩」**没有加引号**,直接写在句里
- 「8」「23」「5」两侧都有半角空格
- 段内不为节奏断行,一段就是一段

### 重点加粗(文字笔记 MUST;配图模式正文可选)

小红书「写文字」支持选中句段后点 **B / 加粗**。

**CRITICAL:正文与加粗分两段输出**

| 输出节 | 内容 | 禁止 |
|---|---|---|
| **§2 正文** | 纯文本,全角标点,可直接复制粘贴 | `**`、`*`、Markdown、HTML |
| **§2b 加粗清单** | 3–6 条,每条抄正文里的原句/词组 | 星号包裹 |

用户流程:先粘贴 §2 正文 → 再按 §2b 清单在 App 内**逐处选中 → B**。

| 规则 | 说明 |
|---|---|
| **用量** | 全文 3–6 处,每处建议 ≤15 字;一段最多 1 处 |
| **可加粗** | 概念对比、转折反高潮、中段评论钩子、实操关键词、情绪短句 |
| **禁止加粗** | 整段、标题复读、干货/必看、连续 3 句以上 |

§2b 示例(无星号):

```
1. 不是转，是备案
2. 只用了四十秒
3. 国家医保服务平台 App
4. 你们家老人最怕的是哪一条
```

### 自检五步(交稿前必做)

1. **搜索 ASCII 半角标点** `,` `.` `;` `:` `?` `!` —— 正文命中即改为全角 `，` `。` `；` `：` `？` `！`
2. 搜索 `**` `*` —— 正文命中即删(加粗只出现在 §2b 清单)
3. 搜索 `"` `"` `「`(只在确实需要时保留),其他全删
4. 找数字 / 英文,确认两侧都有半角空格
5. §2b 加粗清单 3–6 条,且至少 1 条是中段评论钩子
